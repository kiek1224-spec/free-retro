## 1. 프로젝트 소개 (Introduction)
* **Free Retro란?** Free(자유) + Retro(레트로 게임)를 합친 이름으로, OS나 플랫폼의 제약 없이 언제 어디서든 자유롭게 레트로 게임을 즐기자는 목표를 담았습니다.
* **기반 엔진:** C++ 기반 웹어셈블리로 동작하는 `EmulatorJS` 엔진을 활용하여 개발된 웹 기반 에뮬레이터입니다.
* **개발 계기:** 웹 기반 에뮬레이터 서비스인 `Afterplay.io`를 직접 사용해 본 후, *"웹이라는 무대는 정말 자유롭구나!"*라는 충격과 영감을 받아 나만의 커스텀 에뮬레이터를 만들기 시작했습니다.

---

## 2. 주요 특징 (Key Features)

### 🌐 완벽한 크로스 플랫폼 (Cross-Platform)
웹 브라우저를 실행할 수 있는 환경이라면 운영체제에 구애받지 않고 어디서나 작동합니다.
* Windows, Linux, macOS
* Android, iOS (모바일 환경 완벽 지원)

### 📂 유연한 롬파일(ROM) 실행 방식
게임 롬 파일을 플레이하는 방식이 매우 직관적이고 다채롭습니다.
* **로컬 파일:** 브라우저 화면에 롬파일을 드래그 앤 드롭(Drag & Drop)하거나 파일 선택 버튼을 눌러 수동 구동
* **외부 클라우드 연동:** 네이버 MYBOX, 구글 드라이브(Google Drive), 드롭박스(Dropbox) 등 다양한 클라우드 스토리지에 보관된 롬파일을 직접 불러와 실행 가능

## 3. 핵심 기술: 하이브리드 세이브 시스템 (Save System)
`Free Retro`의 가장 큰 기술적 정수이자 독창적인 기능입니다. 에뮬레이터 코어 내부의 `GameManager.js`와 웹 소스(HTML 파일)를 직접 커스텀 개조하여, **로컬 저장과 클라우드(Dropbox) 업로드가 실시간으로 동시에 이루어지는 하이브리드 동기화 구조**를 구현했습니다.

### 💾 지원하는 세이브 종류
1. **State Save (강제 세이브):** 에뮬레이터의 현재 화면과 진행 상태를 실시간 스냅샷 형태로 강제 저장합니다. (`파일명-날짜-시간.state`)
2. **SRM Save (인게임 세이브):** 게임 내부 메뉴(예: 포켓몬스터 게임 내 '리포트 작성')를 통해 저장되는 순수 배터리 세이브 파일입니다. (`파일명.srm` 또는 `파일명-날짜-시간.srm`)

### ⌨️ 조작 단축키 매커니즘
* **`[F1]` 키 (플레이 중 수시 저장):** 누르는 즉시 에뮬레이터의 현재 상태를 `파일명-날짜-시간.state` 포맷으로 드롭박스에 무한 백업합니다. (실수를 방지하는 타임머신 역할)
* **`[ESC]` 키 (게임 종료 및 안전 추출):** 브라우저 종료 시점의 데이터 유실을 완벽히 방지하는 안전장치입니다. 누르는 즉시 종료 시점의 강제 세이브(`.state`)와 인게임 팩 세이브(`.srm`)를 동시에 안전하게 동기화한 후 클라우드로 자동 업로드하고 게임을 종료합니다.

## 4. Dropbox 접근 권한 및 보안 안내 (Security)
본 앱은 서버가 없는 순수 프론트엔드 앱(Static Web)으로, 최소 권한 원칙을 철저히 준수하여 사용자의 소중한 개인정보를 안전하게 보호합니다.

* **접근 범위:** 사용자의 드롭박스 전체 공간이 아닌, 오직 본 앱 전용으로 할당된 **앱 폴더(`Apps/FreeRetro`)만 격리하여 접근**합니다.
* **요구 권한 (Scopes List):**
  * `account_info.read`: 사용자 이름, 이메일, 국가 등 기본 계정 정보 확인
  * `files.metadata.read`: 드롭박스 내 파일 및 폴더 정보 보기
  * `files.metadata.write`: 드롭박스 내 파일 및 폴더 정보 편집
  * `files.content.read`: 세이브 데이터 콘텐츠 불러오기 (다운로드)
  * `files.content.write`: 세이브 데이터 콘텐츠 저장하기 (업로드)
* *⚠️ 주의사항: 안전한 데이터 관리를 위해 앱 폴더(`FreeRetro`) 내부에는 순수 게임 세이브 파일 이외의 다른 개인 파일은 절대 혼용하여 넣지 마세요.*

## 6. 개발 비하인드 (Behind the Scenes)
이 프로젝트는 수많은 시행착오의 연속이었습니다. Gemini AI와 함께 옥타곤에서 스파링하듯 수많은 타이밍 버그 and 비동기 연산 오류를 해결하며 완성한, 이른바 **"멱살잡이 바이브코딩 프로젝트"**입니다. 단순히 구문 코딩을 넘어 웹 브라우저의 가상 파일시스템(IDBFS) 저장 구조, 웹어셈블리 에뮬레이터 코어의 동작 메커니즘, 그리고 클라우드 동기화 타이밍을 깊게 파고들며 이해할 수 있었던 값진 도전이었습니다.
