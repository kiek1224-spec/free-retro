**

**Free Retro**

**소개**

Free Retro는 EmulatorJS 기반으로 제작된 웹 에뮬레이터 프로젝트입니다.

프로젝트 이름은 다음 의미를 담고 있습니다.

* Free = 자유

* Retro = 레트로 게임

운영체제와 환경에 구애받지 않고 자유롭게 레트로 게임을 즐기자는 목표로 개발되었습니다.

**프로젝트를 시작한 계기**

Afterplay와 같은 웹 기반 에뮬레이터 서비스를 사용하면서

"웹이라는 것은 정말 자유롭구나"

라는 생각을 하게 되었고, 그 경험이 Free Retro 프로젝트의 시작점이 되었습니다.

**주요 특징**

**웹 기반 실행**

* 설치 필요 없음

* 브라우저에서 즉시 실행

* 운영체제 무관

* PC 및 모바일 지원

**ROM 파일 실행**

* 드래그 앤 드롭 지원

* 수동 파일 선택 지원

* 자동 기종 인식

지원 기종:

* FC (패미컴)

* SFC (슈퍼 패미컴)

* GB (게임보이)

* GBC (게임보이 컬러)

* GBA (게임보이 어드밴스)

**클라우드 연동**

ROM 파일은 다음 저장소에서 불러올 수 있습니다.

* Dropbox

* Google Drive

* 네이버 MYBOX

Dropbox는 세이브 파일 백업 및 동기화 용도로 사용됩니다.

**세이브 시스템**

Free Retro의 가장 큰 특징입니다.

EmulatorJS 내부 구조와 HTML을 수정하여 구현한 독특한 저장 방식입니다.

**로컬 저장 + 클라우드 업로드**

Free Retro는 다음 두 가지를 동시에 수행합니다.

* 로컬 백업

* Dropbox 업로드

**저장 종류**

**State Save**

강제 저장 방식

파일명-날짜-시간.state

**SRM Save**

게임 내부 저장 방식

파일명.srm

**단축키**

**F1**

* 강제 저장(State)

* 저장 파일 다운로드

**ESC**

* 게임 종료

* State 저장

* SRM 저장

* Dropbox 업로드

**Dropbox 권한**

필요 권한:

* account\_info.read

* files.metadata.read

* files.metadata.write

* files.content.read

* files.content.write

앱 폴더(FreeRetro)만 접근 가능하도록 설정되어 있습니다.

**알려진 버그**

현재 확인된 버그:

* ESC 입력 시 SRM 파일이 중복 다운로드될 수 있음

* ESC 저장 시 State 파일에 타임스탬프가 누락될 수 있음

프로젝트는 완벽함보다 구동을 우선시합니다.

**개발 철학**

이 프로젝트는

"멱살잡이 바이브코딩"

방식으로 개발되었습니다.

완벽한 코드보다 중요한 것은

"실제로 동작하는 결과물"

이라고 생각합니다.
